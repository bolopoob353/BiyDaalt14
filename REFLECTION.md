1. Аль assertion (тест) хамгийн их үнэ цэнэтэй санагдсан вэ? Яагаад?

Schema validation буюу property шалгах assertion хамгийн үнэ цэнэтэй санагдсан.
Учир нь API-ийн response-д заавал байх ёстой талбарууд байгаа эсэхийг автоматаар
шалгадаг. Жишээ нь email талбар байхгүй бол бизнес логик бүтэлгүйтэнэ.

2. Negative test 1+ жишээгээ дэлгэрэнгүй тайлбарла — яг ямар алдааг олох вэ?

Get Invalid Post request-д /posts/999 гэсэн байхгүй id явуулж 404 status
эргэж ирэхийг шалгасан. Энэ тест нь API-ийн алдааны зам зөв ажиллаж байгааг
баталгаажуулдаг. Хэрэглэгч байхгүй resource хайвал API зөв алдааны код буцаах
ёстой.


3. Postman дотор амжилттай ажиллаж байсан тест Newman-д орвол fail болсон уу? Яагаад?

Postman дотор бүх тест pass болж байсан. Newman-д ажиллуулахад эхэндээ
baseUrl хоосон байснаас бүх request fail болсон. env.dev.json файлд утга
зөв оруулсны дараа бүгд pass болсон.



4. Token эсвэл secret-ыг хэрхэн зохицуулсан вэ? (env-ийн соёл)

JSONPlaceholder auth шаардахгүй тул token хэрэглээгүй. Гэхдээ baseUrl-ийг
environment variable-д хадгалснаар dev, staging, prod орчнуудад хялбархан
солих боломжтой болсон.


5. API өөрчлөгдвөл collection-ийн аль хэсэг ХАМГИЙН ИХ эвдрэх вэ? Үүнийг тестийн "эмзэг газар" гэж нэрлэдэг — яаж бууруулах вэ?

Response-ийн property нэр өөрчлөгдвөл schema assertion бүгд эвдэрнэ.
Жишээ нь "email" талбарын нэр "emailAddress" болвол тест fail болно.
Үүнийг бууруулахын тулд assertion-уудыг тусдаа файлд хадгалж,
API өөрчлөгдөх үед зөвхөн тэр файлыг шинэчлэх хэрэгтэй.
